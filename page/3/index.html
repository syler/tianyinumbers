cs 一键买枪bind指令详细教程

对于大多数的CS新手来说，恐怕都得需要小键盘一键买枪，故放出一键买枪的教程。

转载请注明本文来自：http://1few.com/149.html

其实一键买枪只是需要一个bind指令，一般来说，bind指令的格式是这样：

bind "key" "command"

上面列出的都是买得到的枪，如果bind枪，直接像下面这样用：

bind  "buy weapon"

比如说，如果要买AK47，按出来控制台窗口（按1左边那个键，即·）

bind  "buy AK47"

不过一般大家喜欢在一个键上绑定多个命令，比如买了AK之后就自动把主武器弹药加满，或者是按这个键盘还要同时要买上防弹背心和头盔，要达到这种效果，只要用一个分号”;”来分隔不同的命令就行了，例如：

bind  "buy AK47; buy primammo; buy vesthelm"

只要你愿意，你可以用一个键完成很多事情，只要你注意要加分号将这些命令隔开，还要注意，指令得放在一个引号的内部（看我上面的，都有引号）

2. 绑定到小键盘

一般大家都使用小键盘买枪，小键盘能用的有下面的按键

KP_INS (小键盘0) 
KP_DEL (小数点) 
KP_ENTER (Enter键) 
KP_PLUS (加号键) 
KP_END (小键盘1) 
KP_DOWNARROW (小键盘2) 
KP_PGDN (小键盘3) 
KP_LEFTARROW (小键盘4) 
KP_5 (小键盘5) 
KP_RIGHTARROW (小键盘6) 
KP_HOME (小键盘7) 
KP_UPARROW (小键盘8) 
KP_PGUP (小键盘9键) 
KP_SLASH (小键盘 / ) 
* (小键盘 * 键) 
KP_MINUS (小键盘减号)

示例：

code:bind "KP_INS" "buy ak47; buy m4a1; buy primammo" (小键盘0购买AK或M4，看你是警察还是土匪了)

bind "KP_DEL" "buy deagle; buy secammo" (按小数点键买沙鹰和副武器弹药)

bind "KP_LEFTARROW" "buy flashbang" (按4键购买闪光弹)

bind "KP_5" "buy hegrenade" (按5买手雷)

bind "KP_RIGHTARROW" "buy smokegrenade" (按6买烟雾弹)

bind "F3" "buy M4A1; buy AK47; buy primammo; buy flashbang; buy hegrenade; buy defuser" (按F3买M4（或AK）+子弹+一个闪光+一个手雷+一个拆弹器)

小键盘默认是不使用的，所以你可以放心bind，不必担心会影响默认的键位设置。

最末，要说明的是，buy字是可选的，附上我使用的一键买枪bind指令，你只需要复制粘贴执行就行：

log4cxx 使用方法(linux)

下载链接:
http://apr.apache.org/download.cgi
http://logging.apache.org/log4cxx/download.html

第一步安装apr-1.4.5，顺序不能错，它必须首选安装
$tar -zxvf apr-1.4.5.tar.gz
$cd apr-1.4.5
$./configure –prefix=/usr/local/apr
$make
$su root
$make install

然后安装apr-util-1.3.12

$tar -zxvf apr-util-1.3.12.tar.gz
$cd apr-util-1.3.12
$./configure –prefix=/usr/local/apr-util –with-apr=/usr/local/apr
$make
$su root
$make install

configure选项 –with-apr=/usr/local 指定apr库的位置
最后就可以安装log4cxx了
在这之前,必须要改三个文件:
$vim src/examples/cpp/console.cpp 在这个文件里添加#include 和#include
$vim src/main/cpp/socketoutputstream.cpp 在这个文件里添加#include
$vim src/main/cpp/inputstreamreader.cpp 在这个文件里添加#include

$tar -zxvf apache-log4cxx-0.10.0.tar.gz
$cd apache-log4cxx-0.10.0
$./configure –prefix=/usr/local/log4cxx –with-apr=/usr/local/apr –with-apr-util=/usr/local/apr-util

$sudo make
$su root
$make install

最后,在terminal执行:

sudo ldconfig

 

如果

在执行的时候，你可能会得到下面的错误：

error while loading shared libraries: liblog4cxx.so.10: cannot open shared object file: No such file or directory

查找半天发现在/usr/local/lib下有liblog4cxx.so.10文件，这个问题是因为你没有设置LD_LIBRARY_PATH环境变量造成的。

执行

export LD_LIBRARY_PATH=/usr/local/lib

[转]通过Skype和Gtunnel的配合实现翻墙

 

原文来自：http://www.veryfar.com/through-skype-and-gtunne-to-over-the-wall.html

经验证有效，使用Firefox+autoproxy会更爽。

我重新上传了skype网络安装版和GTunnel：

以下为原文：

近来看到很多人在反应，XX门，X界之类的东西，似乎都遇到了一些小问题，虽然我一直是在用VPN，但是对这些软件还是挺有感情了，现在神器不能用了，我们就来自己打造一个出来～

这里用到的炼剑材料如题目说的一样，有Skype和Gtunnel，其中，Skype注意要下载国际版（不是TOM版），这个很多下载站都有的提供的，还有一个Gtunnel，它是一个绿色软件，如果把翻墙的整个过程比喻成一辆汽车的话，Skpype是发动机，而Gtunnel就是将其产生的动力传送到轮胎的装置。由于网上比较难找到Gtunnel，这里提供一个下载链接：

Brsbox：http://www.brsbox.com/filebox/down/fc/464bdc81117ea0e87c8b4c24642759ef

下面说一下使用方法：

1.安装Skype国际版，注册一个帐号，并登陆，然后解压gtunnel-2.0.1.7.7z到任意文件夹，启动它。

2.进入Gtunnel界面，选择Skype模式，这时候Skpye会弹出提示框，询问是否同意Gtunnel的链接，接受即可。

3.有的时候，到第二步已经成功了，不过有时还是会出现一些问题，这里，就需要我们来自己设置一下代理了。

进入IE的Internet选项，选择“连接”选项卡，如果是外网的话，点击“设置”按钮来设置代理，公司使用的是内网，具体设置如图（外网同理），代理地址是127.0.0.1:8081。

代理地址貌似可以在Gtunnel里自行更改…

上面三步完成之后，进入Gtunnel选项，我的设置如图，关闭IE，重新启动Gtunnel，如果进入一个叫“花园网络技术”的网站，那么恭喜你已经成功翻墙了～

如果失败的话，建议检查代理设置～

[原]OpenCV的IPlImage保存为CCITT4压缩的TIFF格式

转载请保留原文地址：http://1few.com/81.html
有一点要知道：提供给 libtiff 和从 libtiff 返回的缓冲区都是每个字节中包含 8 个像素，而8位灰度图是1字节只包含一个像素
OpenCV读入后都是8位灰度图，需要手动进行转换，即将临近的8个像素放入一个字节中，使用位运算即可。

void save(IplImage* img,const char* file)
{//from http://1few.com/81.html
	uint32	width = 0, length = 0, bufsize;
	uint32	nchannels = 1;	
	int16	depth = 1;	
	uint32  rowsperstrip = (uint32) -1;
	uint16	photometric = PHOTOMETRIC_MINISWHITE;
	uint16	config = PLANARCONFIG_CONTIG;
	uint16	fillorder = FILLORDER_MSB2LSB; 
	unsigned char *buf = NULL;
	uint16 compression =COMPRESSION_CCITTFAX4;

	unsigned char *buffer = (unsigned char *)img->imageDataOrigin;
	rowsperstrip = img->height;
	width = img->width;
	length = img->height;
	nchannels = img->nChannels;
	depth = img->depth;

	TIFF	*tiff=TIFFOpen(file,"w");
	TIFFSetField(tiff, TIFFTAG_BITSPERSAMPLE, 1);// set the size of the channels
	TIFFSetField(tiff, TIFFTAG_COMPRESSION, compression);
	TIFFSetField(tiff, TIFFTAG_FILLORDER, fillorder);
	TIFFSetField(tiff, TIFFTAG_IMAGELENGTH, length);
	TIFFSetField(tiff, TIFFTAG_IMAGEWIDTH, width);
	TIFFSetField(tiff, TIFFTAG_ORIENTATION, ORIENTATION_TOPLEFT);
	TIFFSetField(tiff, TIFFTAG_PLANARCONFIG, config);
	TIFFSetField(tiff, TIFFTAG_PHOTOMETRIC, photometric);
	TIFFSetField(tiff, TIFFTAG_RESOLUTIONUNIT, RESUNIT_INCH);
	TIFFSetField(tiff, TIFFTAG_SAMPLEFORMAT, SAMPLEFORMAT_UINT);
	TIFFSetField(tiff, TIFFTAG_SAMPLESPERPIXEL, nchannels);// set number of channels per pixel

	rowsperstrip = TIFFDefaultStripSize(tiff, rowsperstrip);
	if (rowsperstrip > length) {
		rowsperstrip = length;
	}

	TIFFSetField(tiff, TIFFTAG_ROWSPERSTRIP, rowsperstrip );

	int fax_total_cols=0;
	if(width%8==0)
	{
		fax_total_cols=width/8;
	}
	else
	{
		fax_total_cols=width/8+1;
	}

	int fax_current_col=0;
	unsigned char *row=(unsigned char *)_TIFFmalloc(fax_total_cols);
	unsigned int res=0;
	for(int y=0;yheight;y++)
	{
		for(int x=0;xwidth;x++)
		{
			uchar* cp= (uchar*)(img->imageData+img->widthStep*y);
			int pixel=1;
			if(cp[x]==255)
			{
				pixel=0;
			}
			res=res|(pixel<<(7-x%8));
			if((x>0 && x%8==7) || x==img->width-1)
			{
				row[fax_current_col]=res;
				fax_current_col++;
				res=0;
			}
		}

		if(TIFFWriteScanline(tiff,row,y,0)<0)
		{
			break;
		}

		fax_current_col=0;
	}


	if(row)
	{
		_TIFFfree(row);
	}

	TIFFClose(tiff);

}
